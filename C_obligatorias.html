Desarrollo Web en el 2017 + Tips

Presentacion impartida por Adam Carrasco por parte de Oracle


En la conferencia Adam inicio hablando con el frontend y el backend donde:
El frontend son todas aquellas tecnologías que corren del lado del cliente, es decir, todas aquellas tecnologías que corren del lado del navegador web, generalizandose mas que nada en tres lenguajes, Html , CSS Y JavaScript.
El backend es aquel que se encuentra del lado del servidor, es decir, esta persona se encarga de lenguajes como PHP, Python, .Net, Java, etc, es aquel que se encarga de interactuar con bases de datos, verificar manejos de sesiones de usuarios, montar la página en un servidor, y desde este “servir” todas las vistas que el FrontEnd crea, es decir, uno como backend se encarga mas que nada de la manipulación de los datos.

Ademas hablo sobre la importancia de las siguientes Tecnologias.

• Aplicaciones Web Progresivas (PWA)
• Single Web Aplication (SPA)
• Responsive Desing
• Preprosesadores (Css, Scss, Sasss
• Frameworks para el frontend

Tambien Adam enfatizó en aprender estructura de datos y entrenar algoritmos, usar frameworks pero primero enternder como funcionan internamente, Utilizar git para el control de versiones, "Leer Leer Leer" y un punto muy importante "Saber ingles"

Finalizando con el Funcional Testing que es un proceso de garantía de calidad (QA) y un tipo de prueba de caja negra que basa sus casos de prueba en las especificaciones del componente de software bajo prueba. Las funciones se prueban al alimentarlas con entrada y examinar la salida.

Herramientas y técnicas para incrementar la calidad del código

Presentacion impartida por Ing. Eduardo Figarola Mota.


Eduardo en su presentacion inicio con una serie de chistes, muy divertidos. despues de unos minutos, inicio con la presentacion encerio hablando sobre software de calidad y explicando puntos sobre como debe de ser un software de calidad se los dejo en el siguiente parrafo:

• Cumplir un proposito
• Informe cuando un problema ocurre
• Tiene pruebas
• Se entiende su proposito con solo leerlo
• sigue un estilo de codigo
• Utiliza un flujo de colaboracion adecuado
• Implementa correctamente los patrones de diseño
• Se puede utilizar intuitivamente

Despues de explicar cada punto a detalle continuo con sistemas de monitoreo de errores donde menciono a "bugnap, sentry" entre otros.
Continuo con el tipo de pruebas que existen como:
pruebas unitarias: Es una clase sin tener algo externo es decir a un solo documento sin tomar en cuenta clases externas.
pruebas de integración: Son iguales a las anteriores con la diferencia que si tiene encuenta clases externas
pruebas funcionales: Pruebas en la aplicacion Automatizadas.

y para finalizar hablo sobre la Colaboración continua, donde menciono sistemas de control de versiones donde entre ellas estaba git, Subversion y mercurial.
